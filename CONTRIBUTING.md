# Contributing

Please follow these guidelines to collaborate on the project.

Branching
- Use `main` as the protected default branch.
- Create feature branches named `feature/your-short-desc`.

Pull requests
- Open a PR targeting `main` when work is ready for review.
- Add a concise description and link any related issue.
- Keep PRs small and focused.

Code review
- At least one approving review required before merging.

Commits
- Use clear commit messages. Prefix with `feat:`, `fix:`, `chore:`, etc.

Tests & CI
- Ensure CI passes and add tests where applicable.
